🧠 Key technical innovations

Fact-based embeddings – LLM extracts discrete regulatory facts (claim, regulation, article, deadline, action_required, severity) before embedding. Far more precise than chunk-level retrieval.

Two-step hybrid retrieval – keyword pre-filter with industry/country/regulation-specific keywords, then cosine similarity on the filtered subset.

Recency-constrained prompts – all LLM prompts include today's date and a 30-day window. LLM explicitly instructed to ignore historical background and report only recent changes.

Dual-mode output – same pipeline powers both Apify Actor (JSON dataset) and Streamlit frontend (interactive UI with PDF/Markdown export).

🔗 Deep Apify integration

📦 Published Actor

Fully packaged as a reusable Apify Actor with input schema, Dockerfile, and dataset output. Accepts any company profile as JSON input.

🕷️ website-content-crawler

Used for all scraping – EUR-Lex, EDPB, GDPR.eu, and 30+ national regulatory sources. Three scrapers run in parallel via asyncio.gather.

🤖 OpenRouter via Apify Standby

All LLM calls (Claude 3 Haiku) and embeddings (text-embedding-3-small) routed through Apify's OpenRouter Actor – no external API keys needed beyond APIFY_TOKEN.
